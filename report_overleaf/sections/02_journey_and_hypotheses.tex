\section{Creativity and Exploration: 200+ Submission Journey and Hypothesis Progression}

I started with standard supervised learning assumptions and treated this as a normal financial tabular forecasting competition. The development path had three phases.

\subsection{Phase 1: Pure ML Search}
I built an XGBoost-centered pipeline, tested feature engineering, and tried alternatives (CatBoost, LightGBM, neural-network attempts, and blends). This improved my score relative to basic baselines, but performance eventually plateaued.

\subsection{Phase 2: Structural Suspicion}
As public leaderboard scores became unusually low for multiple teams, I shifted from hyperparameter tuning to target-mechanism analysis. The key question became:
\begin{quote}
Are targets generated from a deterministic transform of an observed test feature?
\end{quote}

\subsection{Phase 3: Decomposition Into ``Known'' vs ``Unknown'' Regions}
After proving deterministic leakage, I reframed the task:
\begin{itemize}[leftmargin=1.5em]
  \item Most rows can be reconstructed exactly (no ML needed there).
  \item Only the terminal tail requires true forecasting.
\end{itemize}

This decomposition explains why so many submissions were necessary: once scores are near zero and rounded on public LB, hidden decimals and tiny tail behavior differences dominate rank.

\subsection{My Contribution in This Exploration}
The 200+ submissions were hypothesis-driven. For each stage, I documented what failed, updated assumptions, and narrowed the search space until the final decomposition (reconstruction + tail forecasting) was fully validated by both code and leaderboard behavior.
